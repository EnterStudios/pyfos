pyfos_brocade_logging - PyFOS module to provide rest support for configuring logging facilities in FC switch.¶

The pyfos_brocade_logging provides a REST support for FC switch logging facilities.

class pyfos_brocade_logging.audit(dictvalues={})¶
This class provides methods/options to retrieve and configure
audit logging.

Important class members:

Attribute name Desc. Frequently used methods
audit-enabled Audit Flag peek_audit_enabled() set_audit_enabled()
severity-level Severe Level peek_severity_level() set_severity_level()
filter-class-list /filter-class Class List peek_filter_class_list_filter_class() set_filter_class_list_filter_class()

Object methods

get(session)¶
Returns a :class:`audit` object or a list of objects filled
with attributes gathered from auditcfg.

Each object can be printed using pyfos_util.response_print() and individual attributes accessed through peek methods.

patch(session)¶

Apply configurable attribute(s) within the object to audit

Below is an example using individual sets:

# initialize the audit object
audit_obj = pyfos_brocade_logging.audit()
# set audit enabled to true
audit_obj.set_audit_enabled(true)
# set the severity level to info
audit_obj.set_severity_level(info)
audit_obj.patch(session)

:param session: session handler returned by audit
    :func:`pyfos_auth.login`
:rtype: a dictionary of error or success information

Attribute methods

set_audit_enabled(enable/disable)¶

Sets enabled=true or disabled=false in the object

Parameters:enable/disable – true to enable and false to disable audit
Return type:dictionary of error or success response
peek_audit_enabled()¶

Reads audit enabled state in the object.

Return type:true or false
set_severity_level(level)¶

Sets level of severity in the object

:param : level: Valid severity level: info,warning,critical,error :rtype: dictionary of error or success response

peek_severity_level()¶

Reads severtiy level in the object.

Return type:level of severity of audit
set_filter_class_list_filter_class(filter_class_list)¶

Sets filter class list in the object

:param : level: Valid filter class list - security,configuration, zone,firmware,fabric,ls,cli,maps :rtype: dictionary of error or success response

peek_filter_class_list_filter_class()¶

Reads filter class list in the object.

Return type:filter class list of audit

patch(session)¶

Apply configurable attribute(s) within the object to log_quiet_control.

Below is an example using individual sets:

# initialize the log_quiet_control object
obj = pyfos_brocade_logging.log_quiet_control()
# set the log-type attribute to enable the
# wwn based pid on the log_quiet_control

Below is an example of combining object initialization and attribute sets:

# initialize the log_quiet_control object and
# set the enable attribute
obj = pyfos_brocade_logging.log_quiet_control(

# execute HTTP patch command to apply the object to the
# log_quiet_control connected through session
obj.patch(session)

class pyfos_brocade_logging.syslog_server(dictvalues={})¶
This class provides methods/options to retrieve, configure, modify
and delete syslog.

Important class members:

Attribute name Description Frequently used methods
server syslog server name peek_server() set_server()
port port syslog server peek_port() set_port()
secure-mode secure mode using TLS protocol peek_secure_mode() set_secure_mode()

Object methods

get(session)¶
Returns a :class:`syslog_server` object or list of objects filled with
syslog server attributes.

Each object can be printed using pyfos_util.response_print() and individual attributes accessed through peek methods.

post(session)¶

Add members to a list attribute of the object. This method is used to add syslog servers.

Example usage of the method to configure new syslog server:

# initialize the syslog server object syslog_server_obj = pyfos_brocade_logging.syslog_server() # set the IP address of server syslog_server_obj.set_server(“10.10.10.10”)

Parameters:session – session handler returned by pyfos_auth.login()
Return type:dictionary of error or success response
patch(session)¶

Apply configurable attribute(s) within the object to syslog_server

Below is an example using individual sets:

# initialize the syslog server object
syslog_server_obj = pyfos_brocade_logging.syslog_server()
# set the IP address of server
syslog_server_obj.set_server("10.10.10.10")
# set the port number
syslog_server_obj.set_port("1234")
# set secure mode to true
syslog_server_obj.set_secure_mode(true)
syslog_server_obj.patch(session)

:param session: session handler returned by
    :func:`pyfos_auth.login`
:rtype: a dictionary of error or success information
delete(session)¶

Delete members from a list attribute of the object. This method deletes the syslog server configured.

Example usage of the method to create a new Mappings:

# initialize the syslog server object syslog_server_obj = pyfos_brocade_logging.syslog_server() # set the IP address of server syslog_server_obj.set_server(“10.10.10.10”) # delete the syslog server syslog_server_obj.delete(session)
